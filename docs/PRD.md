# 개인 데일리 블로그 MVP PRD

## 핵심 정보

**목적**: Notion을 CMS로 활용하여 글 작성부터 발행까지 Notion 하나로 관리하는 개인 데일리 블로그
**사용자**: 개발자 본인 및 블로그 방문자 (글을 읽는 일반 독자)

---

## 사용자 여정

```
1. 홈 페이지 (랜딩)
   ↓ 최신 글 카드 탐색

2. 글 목록 페이지
   ↓ 카테고리 탭 선택 또는 검색어 입력

   [카테고리 선택] → 해당 카테고리 글 목록 표시 → 글 카드 클릭
   [검색어 입력]   → 검색 결과 목록 표시       → 글 카드 클릭
   [필터 없음]     → 전체 글 목록 표시          → 글 카드 클릭
   ↓

3. 글 상세 페이지
   ↓ 본문 읽기 완료

4. 완료 → [목록으로 돌아가기] / [브라우저 뒤로가기]
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 글 목록 조회 | Notion DB에서 Status가 발행됨인 글 목록을 가져와 카드 형태로 표시 | 블로그의 핵심 콘텐츠 진입점 | 홈 페이지, 글 목록 페이지 |
| **F002** | 글 상세 조회 | Notion 페이지 블록을 렌더링하여 본문 표시 | 실제 콘텐츠 전달의 핵심 | 글 상세 페이지 |
| **F003** | 카테고리 필터 | Notion DB의 Category 값 기반 탭 필터로 글 목록 좁히기 | 원하는 주제의 글을 빠르게 탐색 | 글 목록 페이지 |
| **F004** | 글 검색 | 제목 기준 클라이언트 사이드 검색 (입력 시 즉시 필터) | 특정 글을 키워드로 직접 찾는 사용자 니즈 | 글 목록 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | Notion API 연동 | Notion 데이터베이스에서 글 데이터 fetch, ISR로 캐싱 | 모든 콘텐츠의 데이터 소스 | 홈 페이지, 글 목록 페이지, 글 상세 페이지 |
| **F011** | Notion 블록 렌더링 | paragraph, heading, image, bulleted_list, numbered_list, quote, divider, code 블록을 React 컴포넌트로 변환 | 본문 표시의 핵심 구현 | 글 상세 페이지 |

### 3. MVP 이후 기능 (제외)

- 댓글 / 반응 기능
- RSS 피드
- 이메일 구독
- 관련 글 추천
- 조회수 카운터
- OG 이미지 자동 생성
- 태그 기반 필터 (카테고리 필터로 대체)

## 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F010` | **메뉴 위치:** 헤더 로고 클릭 / 사이트 루트 접속

| 항목 | 내용 |
|------|------|
| **역할** | 블로그 첫 진입점. 최신 발행 글을 카드 형태로 노출하여 독자 유입 유도 |
| **진입 경로** | 사이트 루트 접속, 헤더 로고 클릭 |
| **사용자 행동** | 최신 글 카드를 훑어보고, 흥미로운 글을 클릭하여 상세 페이지로 이동 |
| **주요 기능** | • 최신 발행 글 6개 카드 그리드 표시 (제목, 카테고리 뱃지, 발행일)<br>• Notion API ISR 캐싱 (revalidate 설정)<br>• **전체 글 보기** 버튼 → 글 목록 페이지 이동<br>• 반응형 카드 그리드 (모바일 1열, 태블릿 2열, 데스크탑 3열) |
| **다음 이동** | 카드 클릭 → 글 상세 페이지, "전체 글 보기" 클릭 → 글 목록 페이지 |

---

### 글 목록 페이지

> **구현 기능:** `F001`, `F003`, `F004`, `F010` | **메뉴 위치:** 헤더 "전체 글"

| 항목 | 내용 |
|------|------|
| **역할** | 발행된 모든 글을 탐색하는 허브. 카테고리 필터와 검색으로 원하는 글을 찾는 공간 |
| **진입 경로** | 헤더 "전체 글" 클릭, 홈 "전체 글 보기" 버튼 클릭 |
| **사용자 행동** | 카테고리 탭으로 주제를 좁히거나, 검색창에 키워드를 입력하여 원하는 글 탐색 |
| **주요 기능** | • 전체 발행 글 카드 그리드 (제목, 카테고리 뱃지, 발행일)<br>• 카테고리 탭 필터 (Notion DB Category 값 기반 동적 생성)<br>• 제목 기준 클라이언트 사이드 검색 (입력 즉시 필터링)<br>• 검색 결과 없을 때 빈 상태(empty state) 메시지 표시<br>• **글 카드 클릭** → 글 상세 페이지 이동 |
| **다음 이동** | 카드 클릭 → 글 상세 페이지 |

---

### 글 상세 페이지

> **구현 기능:** `F002`, `F010`, `F011` | **메뉴 위치:** 홈/글 목록 페이지의 글 카드 클릭 진입

| 항목 | 내용 |
|------|------|
| **역할** | 개별 글 본문을 읽는 페이지. Notion 블록을 React 컴포넌트로 렌더링하여 표시 |
| **진입 경로** | 홈 또는 글 목록 페이지의 글 카드 클릭 |
| **사용자 행동** | 글 본문을 읽고, 읽기 완료 후 목록으로 돌아가거나 브라우저 뒤로가기로 이탈 |
| **주요 기능** | • 글 제목, 카테고리 뱃지, 발행일, 태그 목록 헤더 표시<br>• Notion 블록 렌더링: `paragraph`, `heading_1/2/3`, `image`, `bulleted_list_item`, `numbered_list_item`, `quote`, `divider`, `code`<br>• 페이지 메타데이터 설정 (title, description for SEO)<br>• 존재하지 않는 slug 접근 시 `notFound()` 처리<br>• **목록으로 돌아가기** 링크 → 글 목록 페이지 |
| **다음 이동** | "목록으로 돌아가기" 클릭 → 글 목록 페이지, 브라우저 뒤로가기 |

## 데이터 모델

### Post (Notion DB 매핑)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Notion 페이지 ID | string (UUID) |
| title | 글 제목 | string |
| category | 카테고리 | string |
| tags | 태그 목록 | string[] |
| publishedAt | 발행일 | Date |
| status | 상태 | "draft" \| "published" |
| slug | URL 식별자 (Notion page id 기반) | string |

### NotionBlock (렌더링용)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 블록 ID | string |
| type | 블록 타입 (paragraph, heading_1 등) | string |
| content | 렌더링할 텍스트 또는 미디어 데이터 | object |
| children | 하위 블록 (중첩 리스트 등) | NotionBlock[] |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16** (App Router) - React 풀스택 프레임워크, ISR로 정적 페이지 생성
- **TypeScript 5.x** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** (`app/globals.css`의 `@theme` 블록으로 CSS 변수 관리, `tailwind.config.js` 없음)
- **shadcn/ui** (radix-nova 스타일) - 고품질 React 컴포넌트
- **Lucide React** - 아이콘

### CMS 연동

- **@notionhq/client** - Notion 공식 API 클라이언트
- **ISR (Incremental Static Regeneration)** - `revalidate` 설정으로 빌드 후 주기적 재생성

### 배포 & 환경 변수

- **Vercel** - Next.js 최적화 배포 플랫폼
- `.env.local` 환경 변수:
  ```
  NOTION_API_KEY=secret_xxx
  NOTION_DATABASE_ID=xxx
  ```
