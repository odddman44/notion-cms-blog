## 시나리오 2: 카테고리 필터 + 검색 조합

**사전 조건:** "개발" 카테고리 글 1건 이상

| 단계 | 액션 | 기대값 | 결과 |
|---|---|---|---|
| 1 | `/posts` 접속 | "전체"/"개발" 필터 버튼 표시, 전체 글 노출 | ✅ |
| 2 | "개발" 카테고리 버튼 클릭 | 버튼에 `bg-accent` 클래스(active), "개발" 글만 표시 | ✅ |
| 3 | 검색어 "Notion" 입력 + 300ms 대기 | "개발" + "Notion" 복합 필터, 매칭 글 표시 | ✅ |
| 4 | 검색어 "없는글제목xyz" 입력 + 500ms 대기 | 카드 0개, "검색 결과가 없습니다." 메시지 | ✅ |

## 시나리오 3: 빈 상태 및 404 처리

| 단계 | 액션 | 기대값 | 결과 |
|---|---|---|---|
| 1 | 존재하지 않는 검색어 입력 | "검색 결과가 없습니다." empty state 표시 | ✅ |
| 2 | `/posts/invalid-slug-that-does-not-exist` 접속 | "페이지를 찾을 수 없습니다" 404 페이지, "홈으로 돌아가기" 버튼 | ✅ |

> **ISR revalidate 동작 확인 방법**: 프로덕션 빌드(`npm run build && npm start`)에서 Notion에 새 글 발행 후 1시간(`revalidate=3600`) 이내에는 이전 캐시, 이후에는 새 글 노출 여부 확인. 로컬 dev 환경에서는 캐싱 없이 매 요청마다 fetch하므로 dev 서버에서 직접 검증 불가.

## 시나리오 4: 반응형 레이아웃 (모바일 / 태블릿 / 데스크탑)

**Playwright `browser_resize`로 뷰포트 전환**

| 뷰포트 | 너비 | 데스크탑 nav | 햄버거 버튼 | 결과 |
|---|---|---|---|---|
| 모바일 | 375px | `display: none` | `display: flex` ("메뉴 열기") | ✅ |
| 태블릿 (경계) | 768px | `display: flex` | `display: none` | ✅ |
| 데스크탑 | 1280px | `display: flex` | DOM에서 제거됨 | ✅ |

- md 브레이크포인트(768px)에서 데스크탑 nav가 정확히 활성화됨
- 모바일에서는 햄버거("메뉴 열기") 버튼이 Sheet 기반 드로어로 노출됨
